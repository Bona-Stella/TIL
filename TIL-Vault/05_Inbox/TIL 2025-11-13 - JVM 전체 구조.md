## **1. 🏗️ JVM의 4가지 주요 구성 요소 (구조)**

JVM은 크게 **클래스 로더 시스템, 런타임 데이터 영역, 실행 엔진, Native 인터페이스 및 라이브러리**로 구성됩니다.

### **A. 클래스 로더 시스템 (Class Loader System)**

| **역할**    | **설명**                                                                                   |
| --------- | ---------------------------------------------------------------------------------------- |
| **변환**    | `.class` 파일(바이트코드)을 JVM이 사용할 수 있도록 **메모리(Runtime Data Area)**로 로드하는 역할을 합니다.             |
| **핵심 단계** | **로딩(Loading)** $\to$ **링킹(Linking)** $\to$ **초기화(Initialization)**의 3단계를 거쳐 클래스를 준비합니다. |
### **B. 런타임 데이터 영역 (Runtime Data Area)**

JVM이 프로그램을 실행하는 데 사용하는 **메모리 영역**입니다. 이 영역은 **스레드 공유 영역**과 **스레드 전용 영역**으로 나뉩니다.

| **영역**                    | **공유 여부** | **역할**                                                                             |
| ------------------------- | --------- | ---------------------------------------------------------------------------------- |
| **메서드 영역 (Method Area)**  | **공유**    | 클래스 및 메서드 정보, static 변수, 상수 풀 등 **클래스 단위 정보**를 저장합니다. (Java 8부터 **메타스페이스**로 대체됨)   |
| **힙 영역 (Heap Area)**      | **공유**    | **`new` 키워드로 생성된 객체 인스턴스**와 **배열**을 저장합니다. **GC(Garbage Collection)**의 주된 대상이 됩니다. |
| **JVM 스택 (JVM Stack)**    | **전용**    | **메서드가 호출될 때마다** 해당 스레드에 할당되며, **지역 변수, 매개 변수, 리턴 주소** 등을 저장합니다.                   |
| **PC 레지스터 (PC Register)** | **전용**    | 현재 스레드가 실행할 **JVM 명령어의 주소**를 저장합니다. 스레드마다 어떤 명령을 실행할지 추적합니다.                       |
| **네이티브 메서드 스택**           | **전용**    | 자바 코드가 아닌 **C/C++ 코드**로 작성된 **Native 메서드**를 호출할 때 사용되는 스택입니다.                      |

### **C. 실행 엔진 (Execution Engine)**

클래스 로더가 로드한 바이트코드를 실행하는 핵심 구성 요소입니다.

|**구성 요소**|**역할**|
|---|---|
|**인터프리터**|바이트코드를 **한 줄씩 읽어** 기계어로 변환하고 실행합니다. 초기 실행에 사용되지만 느립니다.|
|**JIT 컴파일러**|자주 사용되는 코드(핫스팟)를 **네이티브 기계어**로 컴파일하여 캐시에 저장 후 재사용합니다. 실행 속도를 높입니다.|
|**GC (Garbage Collector)**|힙 영역에서 더 이상 사용되지 않는 객체를 찾아 제거하는 역할을 합니다.|

### **D. Java Native Interface (JNI) & Native Method Libraries**

**JVM 외부에 있는 OS 자원**이나 **Native 코드(C/C++)**와 상호작용하기 위한 인터페이스입니다. 이를 통해 자바가 플랫폼 종속적인 기능을 사용할 수 있게 됩니다.

## **2. 🚀 Java의 하이브리드 실행 방식**

말씀하신 것처럼 Java의 실행 방식은 **컴파일**과 **인터프리트**가 결합된 하이브리드 형태입니다.

1. **컴파일:** 개발자가 작성한 `.java` 파일을 **Java 컴파일러**가 **플랫폼 독립적인 바이트코드(.class)**로 변환합니다.
    
2. **로드:** **클래스 로더**가 바이트코드를 런타임 데이터 영역으로 가져옵니다.
    
3. **실행 (하이브리드):** **실행 엔진**이 바이트코드를 처리합니다.
    
    - **인터프리터:** 코드를 처음 실행할 때 한 줄씩 기계어로 번역하여 실행합니다.
        
    - **JIT 컴파일러:** 자주 실행되는 코드는 인터프리트하지 않고 **기계어로 컴파일하여 캐시**한 후 고속으로 실행합니다.
        

이 방식 덕분에 Java는 플랫폼 독립성을 유지하면서도 높은 성능을 달성할 수 있습니다.

# 📝 TIL 2025-11-14 - IoC 컨테이너

> **관련 주제:** [[MOC_TBD]] 

## 🚀 개요 및 핵심 (Summary)

IoC 컨테이너는 단순한 주입 기능을 넘어 **객체 풀(Pool) 관리, 생명주기 관리, 의존성 해결, 그리고 프레임워크 기능(AOP 등) 적용**을 전담하여 개발자가 비즈니스 로직에만 집중할 수 있도록 환경을 조성하는 역할

## 📚 상세 내용 (Deep Dive)

## 🛠️ Spring IoC 컨테이너의 주요 역할 ⚙️

Spring의 IoC 컨테이너(주로 `ApplicationContext`)는 애플리케이션의 **모든 핵심 객체(Bean)의 중앙 관리자** 역할을 합니다.

### 1. 🧩 객체의 생성 및 관리 (Life Cycle Management)

- **Bean 생성:** 개발자가 지정한 설정 정보(`@Component`, `@Bean` 등)를 기반으로 **런타임 시**에 객체(Bean)를 생성합니다.
    
- **스코프 관리:** Bean의 **스코프(Scope)**에 따라 싱글톤, 프로토타입, 요청(Request) 단위 등으로 객체의 수와 수명 주기(생성 $\to$ 초기화 $\to$ 소멸)를 관리합니다.
    
- **초기화/소멸:** `@PostConstruct`, `@PreDestroy` 등의 생명주기 콜백을 호출하여, 객체 사용 전후에 필요한 로직(예: 자원 초기화, 연결 해제)을 자동으로 실행합니다.
    

### 2. 🔗 의존성 주입 (DI, Dependency Injection)

- **책임의 분리:** 개발자가 직접 `new` 키워드로 객체를 생성하고 의존성을 연결하는 대신, 컨테이너가 `@Autowired` 등의 정보를 기반으로 객체 간의 **의존 관계를 자동으로 연결(주입)**해 줍니다.
    
- **느슨한 결합:** 이 DI 기능을 통해 객체들은 서로의 구체적인 구현체에 직접 의존하지 않고, **추상화(인터페이스)**에 의존하는 **느슨한 결합**을 유지할 수 있게 됩니다.
    

### 3. 🛡️ AOP(관점 지향 프로그래밍) 지원

- **프록시 생성:** 컨테이너는 Bean을 생성할 때, AOP 설정을 확인하여 해당 Bean에 **트랜잭션**이나 **보안, 로깅** 등의 횡단 관심사(Cross-Cutting Concerns)를 적용해야 할 경우, **동적으로 프록시 객체**를 생성하고 주입합니다.

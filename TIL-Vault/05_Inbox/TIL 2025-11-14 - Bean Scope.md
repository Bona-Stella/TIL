# 📝 TIL 2025-11-14 - Bean Scope

> **관련 주제:** [[MOC_TBD]] 

## 🚀 개요 및 핵심 (Summary)

Spring Bean의 **Scope**는 Spring IoC 컨테이너가 **Bean 객체를 언제, 몇 개나 생성하고 관리할지**를 정의하는 범위

Bean 생성 시점은 **컴파일 시**가 아니라 **런타임 시(애플리케이션 구동 시 또는 HTTP 요청 시)**에 결정

---

## 📚 상세 내용 (Deep Dive)

## ☕ Spring Bean Scope의 종류

Spring은 기본적으로 6가지 Scope를 제공하며, 크게 **싱글톤(기본)**과 **프로토타입**의 두 가지 주요 범주로 나눌 수 있습니다.

### 1. ⚙️ 싱글톤 계열 (가장 많이 사용)

|**Scope**|**키워드**|**생성 및 관리 시점**|**설명**|
|---|---|---|---|
|**Singleton**|`singleton` (기본값)|**컨테이너 구동 시**|컨테이너 내에 **단 하나의 객체**만 생성되어 관리됩니다. 모든 요청과 주입 시 이 하나의 인스턴스를 공유합니다. **Thread-Safe를 보장하지 않으므로** 공유 상태(데이터)가 없어야 합니다.|
|**Request**|`request`|**HTTP 요청 시**|**하나의 HTTP Request** 동안만 유지되며, 요청이 끝나면 소멸됩니다. 주로 웹 환경에서 요청별 고유한 상태를 저장할 때 사용됩니다.|
|**Session**|`session`|**HTTP Session 생성 시**|**하나의 HTTP Session** 동안만 유지되며, 세션이 종료되면 소멸됩니다. 주로 사용자 세션별 고유한 상태(예: 로그인 정보, 장바구니)를 저장할 때 사용됩니다.|
|**Application**|`application`|**ServletContext 생성 시**|웹 애플리케이션 전체(서버)에서 단 하나만 생성되어 공유되며, 서블릿 컨텍스트가 종료될 때까지 유지됩니다. 웹 환경의 싱글톤과 유사합니다.|

Spring Bean의 **Scope**는 Spring IoC 컨테이너가 **Bean 객체를 언제, 몇 개나 생성하고 관리할지**를 정의하는 범위입니다.

말씀하신 것처럼 Bean 생성 시점은 **컴파일 시**가 아니라 **런타임 시(애플리케이션 구동 시 또는 HTTP 요청 시)**에 결정됩니다. 🛠️

### 2. 🧩 프로토타입 계열

|**Scope**|**키워드**|**생성 및 관리 시점**|**설명**|
|---|---|---|---|
|**Prototype**|`prototype`|**Bean을 주입받거나 요청할 때마다**|Bean을 주입받거나 `getBean()` 메서드가 호출될 때마다 **항상 새로운 객체**가 생성됩니다. Spring 컨테이너가 초기화까지만 관여하며, 이후 소멸 관리는 개발자에게 맡겨집니다.|
|**WebSocket**|`websocket`|**WebSocket 연결 동안**|WebSocket 환경에서 하나의 WebSocket 세션 동안만 유지됩니다.|
